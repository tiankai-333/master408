# 数据库字符编码问题修复指导

## 问题描述

登录后学科名称显示为 "????"，表示数据库字符编码存在问题。

## 问题根因

数据库连接配置 `application-dev.yml` 中的字符集设置与数据库实际字符集不匹配。

## 修复步骤

### 步骤1：检查 MySQL 数据库字符集配置

登录 MySQL 执行以下命令：

```sql
-- 查看数据库字符集
SHOW CREATE DATABASE xzs;

-- 查看表字符集
SHOW CREATE TABLE t_subject;

-- 查看连接字符集
SHOW VARIABLES LIKE 'character%';
SHOW VARIABLES LIKE 'collation%';
```

### 步骤2：根据检查结果修复

**如果数据库是 utf8mb4：**

```sql
-- 确保数据库使用 utf8mb4
ALTER DATABASE xzs CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

**如果需要修复表和字段：**

```sql
-- 修复表的字符集
ALTER TABLE t_subject CONVERT TO CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
ALTER TABLE t_question CONVERT TO CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
ALTER TABLE t_text_content CONVERT TO CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
ALTER TABLE t_user CONVERT TO CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
-- 继续修复其他表...
```

### 步骤3：重新导入数据

如果数据已经损坏，需要重新导入：

```sql
-- 先备份现有数据（重要！）
-- 然后执行 master408.sql 重新导入学科和题目数据
```

### 步骤4：验证修复

```sql
-- 验证学科数据
SELECT id, name, level, level_name FROM t_subject WHERE deleted=0;

-- 应该显示：
-- | id | name | level | level_name |
-- |----|------|-------|------------|
-- |100 | 数据结构 | 1 | 期末考 |
-- |101 | 数据结构 | 2 | 考研 |
-- ...
```

## 预防措施

1. 定期备份数据库
2. 导入 SQL 文件时确保文件编码为 UTF-8
3. MySQL 连接配置使用 `utf8mb4` 而非 `utf8`
4. 确保数据库、表、字段的字符集一致

## 相关配置文件

- 数据库配置：`source/xzs/src/main/resources/application-dev.yml`
- 数据库脚本：`sql/xzs-mysql.sql` 和 `sql/master408.sql`

## 注意事项

- 执行 ALTER TABLE 会锁定表，生产环境请谨慎操作
- 建议在低峰期执行字符集转换
- 转换前务必备份数据
