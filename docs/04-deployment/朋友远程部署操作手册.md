# 408Master 远程部署操作手册

适用场景：项目负责人当前网络无法 SSH 到云服务器，但朋友的网络可以连接服务器。本手册给朋友照步骤执行。

> 安全提醒：本文不写明服务器密码。请项目负责人通过单独渠道把 root 密码发给执行部署的人。部署完成后建议修改密码或改用 SSH key。

## 1. 目标和服务器信息

| 项目 | 值 |
|---|---|
| 服务器公网 IP | `118.31.34.132` |
| SSH 用户 | `root` |
| SSH 端口 | `22` |

| 推荐部署分支 | `codex/ai-knowledge-rag` |
| 对外访问端口 | `80` |

最终访问地址：

## 2. 朋友先确认能连上服务器

在朋友电脑上执行：

```bash
ssh root@118.31.34.132
```

如果连接超时，说明朋友当前网络也不在白名单内，或服务器安全组没有放行 22 端口。需要在云平台安全组里放行朋友公网 IP 的 TCP 22。

如果能进入服务器，先检查基础环境：

```bash
uname -a
docker --version
docker compose version || docker-compose --version
df -h
free -h
```

如果没有 Docker，Ubuntu 可以这样安装：

```bash
apt update
apt install -y ca-certificates curl gnupg git rsync
curl -fsSL https://get.docker.com | bash
systemctl enable --now docker
docker --version
```

## 3. 在朋友电脑准备部署包

朋友可以在自己的电脑上 clone 代码、构建前后端，再上传构建产物。这样比在 20GB 小服务器上编译更稳。

```bash
git clone https://github.com/tiankai-333/master408.git
cd master408
git fetch origin
git checkout codex/ai-knowledge-rag
git pull origin codex/ai-knowledge-rag
```

### 3.1 构建学生端

```bash
cd source/vue/xzs-student
npm install --registry=https://registry.npmmirror.com
npm run build
cd ../../..
```

### 3.2 构建管理端

管理端当前 `.env` 里可能是 `http://localhost:8000`。线上部署时浏览器里的 `localhost` 会指向访问者自己的电脑，所以构建前要覆盖为同源请求。

```bash
cd source/vue/xzs-admin
printf 'VITE_APP_URL=\n' > .env.production
npm install --registry=https://registry.npmmirror.com
npm run build
cd ../../..
```

### 3.3 构建后端

```bash
cd source/xzs
mvn clean package -DskipTests
cd ../..
```

确认产物存在：

```bash
ls -lh source/xzs/target/xzs-3.9.0.jar
ls source/vue/xzs-student/dist/index.html
ls source/vue/xzs-admin/dist/index.html
```

## 4. 上传部署文件到服务器

在朋友电脑的项目根目录执行：

```bash
ssh root@118.31.34.132 "mkdir -p /opt/xzs-deploy/sql /opt/xzs-deploy/static/student /opt/xzs-deploy/static/admin /opt/xzs-deploy/logs"

scp deploy/Dockerfile deploy/docker-compose.yml deploy/nginx.conf root@118.31.34.132:/opt/xzs-deploy/
scp source/xzs/target/xzs-3.9.0.jar root@118.31.34.132:/opt/xzs-deploy/

rsync -av --delete source/vue/xzs-student/dist/ root@118.31.34.132:/opt/xzs-deploy/static/student/
rsync -av --delete source/vue/xzs-admin/dist/ root@118.31.34.132:/opt/xzs-deploy/static/admin/

scp database/current/01_init_structure.sql \
    database/current/02_extend_fields.sql \
    database/current/04_exam_data.sql \
    database/current/05_rag_embeddings.sql \
    database/current/06_ai_knowledge_rag.sql \
    root@118.31.34.132:/opt/xzs-deploy/sql/

rsync -av crawler/ root@118.31.34.132:/opt/xzs-deploy/crawler/
```

## 5. 在服务器启动 Docker 服务

SSH 进入服务器：

```bash
ssh root@118.31.34.132
cd /opt/xzs-deploy
```

如果是第一次部署，或者允许清空旧数据库，建议备份旧数据目录后重新初始化：

```bash
docker compose down || true
if [ -d mysql-data ]; then mv mysql-data "mysql-data.bak.$(date +%Y%m%d%H%M%S)"; fi
```

启动 MySQL：

```bash
docker compose up -d mysql
sleep 20
docker logs xzs-mysql --tail 50
```

确认 MySQL 能连接：

```bash
docker exec xzs-mysql mysql -u root -p'doushijiaxiang0.' -e "SELECT VERSION();"
```

## 6. 按顺序导入数据库

不要依赖 Docker 自动导入多个 SQL 文件，因为文件名顺序不一定等于项目需要的导入顺序。手动执行下面命令：

```bash
cd /opt/xzs-deploy

docker exec xzs-mysql mysql -u root -p'doushijiaxiang0.' -e \
  "CREATE DATABASE IF NOT EXISTS xzs DEFAULT CHARSET utf8mb4 COLLATE utf8mb4_unicode_ci;"

for f in \
  01_init_structure.sql \
  02_extend_fields.sql \
  04_exam_data.sql \
  05_rag_embeddings.sql \
  06_ai_knowledge_rag.sql
do
  echo "IMPORT $f"
  docker exec -i xzs-mysql mysql -u root -p'doushijiaxiang0.' xzs < "sql/$f"
done
```

验证基础数据：

```bash
docker exec xzs-mysql mysql -u root -p'doushijiaxiang0.' xzs -e "
SELECT 't_text_content' AS tbl, COUNT(*) AS cnt FROM t_text_content
UNION ALL SELECT 't_question', COUNT(*) FROM t_question
UNION ALL SELECT 't_exam_paper', COUNT(*) FROM t_exam_paper
UNION ALL SELECT 't_subject', COUNT(*) FROM t_subject
UNION ALL SELECT 't_ai_knowledge_base', COUNT(*) FROM t_ai_knowledge_base;
"
```

## 7. 导入 408 知识点爬虫数据

这一步用于把 116 条 408 知识点写入 `knowledge_point` 和 `t_ai_knowledge_base`，是 RAG/Skill 亮点的重要数据基础。

为了让服务器宿主机上的 Python 能访问 Docker MySQL，需要临时把 MySQL 绑定到本机 127.0.0.1。编辑 `/opt/xzs-deploy/docker-compose.yml`，在 `mysql` 服务下加入：

```yaml
    ports:
      - "127.0.0.1:3306:3306"
```

然后重启 MySQL：

```bash
cd /opt/xzs-deploy
docker compose up -d mysql
```

安装 Python 依赖并导入快照：

```bash
apt install -y python3 python3-pip
pip3 install -r crawler/requirements.txt pymysql

python3 crawler/knowledge_crawler.py \
  --skip-crawl \
  --import-db \
  --clear-existing \
  --db-host 127.0.0.1 \
  --db-port 3306 \
  --db-user root \
  --db-password 'doushijiaxiang0.' \
  --db-name xzs \
  --output crawler/data/knowledge_pages.json
```

验证知识库数量：

```bash
docker exec xzs-mysql mysql -u root -p'doushijiaxiang0.' xzs -e "
SELECT source_type, COUNT(*) FROM t_ai_knowledge_base GROUP BY source_type;
SELECT COUNT(*) AS knowledge_points FROM knowledge_point;
"
```

预期至少看到：

- `csgraduates` 约 116 条
- `method_summary` 约 3 条

## 10. 常见问题排查

### 10.1 SSH 超时

这不是密码问题，是网络或安全组问题。检查：

- 云平台安全组入方向是否放行 TCP 22
- 是否只允许朋友 IP 访问
- 服务器是否开机
- SSH 端口是否改过

### 10.2 页面打开但接口报错

查看后端日志：

```bash
docker logs xzs-backend --tail 100
```

查看数据库是否有数据：

```bash
docker exec xzs-mysql mysql -u root -p'doushijiaxiang0.' xzs -e "
SELECT COUNT(*) FROM t_question;
SELECT COUNT(*) FROM t_exam_paper;
SELECT COUNT(*) FROM t_ai_knowledge_base;
"
```

### 10.3 Nginx 502

通常是后端没起来：

```bash
docker ps | grep xzs-backend
docker logs xzs-backend --tail 100
```

### 10.4 管理端请求打到 localhost

说明管理端构建时 `VITE_APP_URL` 没有改掉。回到朋友电脑重新执行：

```bash
cd source/vue/xzs-admin
printf 'VITE_APP_URL=\n' > .env.production
npm run build
rsync -av --delete dist/ root@118.31.34.132:/opt/xzs-deploy/static/admin/
```

然后浏览器强制刷新。

### 10.5 AI 分析不可用

检查后端配置里的大模型 API key 是否有效。能打开系统不代表 AI 调用一定可用，AI 调用还依赖第三方模型服务。

```bash
docker logs xzs-backend --tail 200 | grep -i -E "ai|glm|openai|error|exception"
```
