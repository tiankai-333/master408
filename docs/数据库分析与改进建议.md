## 二、现有表的问题分析

### 2.1 字符集问题（高优先级）
**问题**：
- 表定义使用 `utf8mb4_general_ci` 排序规则
- 建议统一改为 `utf8mb4_unicode_ci` 以获得更好的多语言排序支持

**改进建议**：
```sql
-- 检查当前字符集
SELECT TABLE_NAME, TABLE_COLLATION 
FROM information_schema.TABLES 
WHERE TABLE_SCHEMA = 'xzs';

-- 统一修改为 utf8mb4_unicode_ci
ALTER TABLE t_subject CONVERT TO CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
-- 对其他表执行相同操作
```

---

### 2.2 索引缺失问题（中优先级）

#### 问题1：t_question 表缺少必要索引
```sql
-- 现有索引：PRIMARY KEY (id)
-- 缺少以下索引：
ALTER TABLE t_question ADD INDEX idx_subject_id (subject_id);
ALTER TABLE t_question ADD INDEX idx_grade_level (grade_level);
ALTER TABLE t_question ADD INDEX idx_question_type (question_type);
ALTER TABLE t_question ADD INDEX idx_status_deleted (status, deleted);
ALTER TABLE t_question ADD INDEX idx_create_time (create_time);
```

**原因**：查询题目列表、按学科/年级筛选时无索引支持，全表扫描影响性能。

#### 问题2：t_exam_paper 表缺少索引
```sql
-- 建议添加索引
ALTER TABLE t_exam_paper ADD INDEX idx_subject_id (subject_id);
ALTER TABLE t_exam_paper ADD INDEX idx_grade_level (grade_level);
ALTER TABLE t_exam_paper ADD INDEX idx_paper_type (paper_type);
ALTER TABLE t_exam_paper ADD INDEX idx_create_user (create_user);
ALTER TABLE t_exam_paper ADD INDEX idx_deleted_status (deleted, status);
```

#### 问题3：t_user_event_log 表缺少索引
```sql
-- 建议添加索引
ALTER TABLE t_user_event_log ADD INDEX idx_user_id (user_id);
ALTER TABLE t_user_event_log ADD INDEX idx_create_time (create_time);
```

---

### 2.3 数据冗余问题（中优先级）

#### 问题1：t_exam_paper_answer 表冗余字段
**当前设计**：
- `paper_name` - 冗余存储试卷名称
- `subject_id` - 冗余存储学科ID

**改进建议**：
- 通过 `exam_paper_id` 关联查询试卷表获取完整信息
- 或保留冗余字段但增加数据同步机制

#### 问题2：t_exam_paper_question_customer_answer 表冗余
**当前设计**：
- `subject_id` - 冗余存储学科ID
- `question_score` - 冗余存储题目分数

**改进建议**：
- 可以通过 `question_id` 关联查询题目表获取这些信息
- 但考虑到性能，可以在查询频率高时保留冗余

---

### 2.4 字段设计问题（中优先级）

#### 问题1：t_user 表字段设计
**缺失字段**：
```sql
-- 建议添加以下字段
ALTER TABLE t_user 
ADD COLUMN email VARCHAR(100) COMMENT '邮箱',
ADD COLUMN last_login_ip VARCHAR(50) COMMENT '最后登录IP',
ADD COLUMN last_login_time DATETIME COMMENT '最后登录时间',
ADD COLUMN login_count INT DEFAULT 0 COMMENT '登录次数',
ADD COLUMN extend_info TEXT COMMENT '扩展信息（JSON格式）';
```

**密码字段问题**：
- 当前混合存储 RSA加密 和 BCrypt 格式
- 建议统一迁移到 BCrypt 格式

#### 问题2：t_subject 表可优化
**当前问题**：
- `level` 和 `level_name` 混合存储在一个表

**改进建议**：
- 考虑将考试类型（期末考、考研、复试）单独建表
- 当前设计可以通过 level 字段关联

---

### 2.5 软删除设计不统一（低优先级）

#### 问题1：deleted 字段类型不一致
**当前问题**：
- 大多数表使用 `bit(1)` 类型
- AI相关表使用 `tinyint(1)`

**改进建议**：
```sql
-- 统一使用 tinyint(1)
ALTER TABLE t_user MODIFY COLUMN deleted TINYINT(1) DEFAULT 0;
ALTER TABLE t_question MODIFY COLUMN deleted TINYINT(1) DEFAULT 0;
ALTER TABLE t_subject MODIFY COLUMN deleted TINYINT(1) DEFAULT 0;
-- 对其他表执行相同操作
```

#### 问题2：缺少统一删除时间字段
```sql
-- 建议添加删除时间字段
ALTER TABLE t_user ADD COLUMN delete_time DATETIME COMMENT '删除时间';
ALTER TABLE t_question ADD COLUMN delete_time DATETIME COMMENT '删除时间';
-- 用于数据恢复和审计
```

## 三、AI扩展表分析与建议

### 3.1 t_ai_usage_log 表优化（高优先级）

**当前问题**：
- 使用 TEXT 字段存储 `question` 和 `response`
- 日志表增长快，可能影响查询性能

**改进建议**：
```sql
-- 添加分区支持（按时间分区）
ALTER TABLE t_ai_usage_log PARTITION BY RANGE (YEAR(create_time) * 100 + MONTH(create_time)) (
    PARTITION p202501 VALUES LESS THAN (202502),
    PARTITION p202502 VALUES LESS THAN (202503),
    PARTITION p202503 VALUES LESS THAN (202504),
    -- 更多分区...
);

-- 或者使用 MySQL 8.0+ 的自动分区
```

**表结构优化建议**：
```sql
-- 建议将长文本字段拆分
CREATE TABLE t_ai_usage_log (
    `id` INT(11) NOT NULL AUTO_INCREMENT,
    `template_id` INT(11) DEFAULT NULL,
    `style` VARCHAR(50) NOT NULL,
    `question_short` VARCHAR(500) COMMENT '问题摘要（前500字符）',
    `question_full_id` INT(11) COMMENT '关联完整问题表',
    -- 其他字段...
    PRIMARY KEY (`id`),
    KEY `idx_template_id` (`template_id`),
    KEY `idx_style` (`style`),
    KEY `idx_create_time` (`create_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 单独存储完整问答记录
CREATE TABLE t_ai_usage_content (
    `id` INT(11) NOT NULL AUTO_INCREMENT,
    `usage_log_id` INT(11) NOT NULL,
    `question` LONGTEXT,
    `prompt` LONGTEXT,
    `response` LONGTEXT,
    PRIMARY KEY (`id`),
    KEY `idx_usage_log_id` (`usage_log_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 五、数据安全与审计

### 5.1 敏感数据加密（中优先级）

**建议**：
```sql
-- 密码字段使用 BCrypt（已完成）
-- API密钥使用 AES 加密
ALTER TABLE t_ai_model_config MODIFY COLUMN api_key VARBINARY(255);

-- 用户IP等信息加密存储
ALTER TABLE t_user ADD COLUMN last_login_ip_encrypted VARBINARY(100);
```

---

### 5.2 审计日志增强（低优先级）

**建议添加操作审计表**：
```sql
CREATE TABLE t_audit_log (
    `id` BIGINT NOT NULL AUTO_INCREMENT,
    `user_id` INT COMMENT '操作用户',
    `action` VARCHAR(50) NOT NULL COMMENT '操作类型',
    `table_name` VARCHAR(50) COMMENT '操作的表',
    `record_id` INT COMMENT '操作的记录ID',
    `old_value` JSON COMMENT '修改前的值',
    `new_value` JSON COMMENT '修改后的值',
    `ip_address` VARCHAR(50) COMMENT 'IP地址',
    `user_agent` VARCHAR(255) COMMENT '用户代理',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    KEY `idx_user_id` (`user_id`),
    KEY `idx_action` (`action`),
    KEY `idx_create_time` (`create_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='操作审计日志';
```

## 附录：快速诊断SQL

```sql
-- 1. 检查表大小
SELECT 
    table_name AS '表名',
    ROUND(data_length / 1024 / 1024, 2) AS '数据大小(MB)',
    ROUND(index_length / 1024 / 1024, 2) AS '索引大小(MB)'
FROM information_schema.TABLES 
WHERE table_schema = 'xzs'
ORDER BY data_length DESC;

-- 2. 检查缺失索引
SELECT 
    table_name,
    index_name,
    non_unique,
    seq_in_index,
    column_name
FROM information_schema.STATISTICS
WHERE table_schema = 'xzs'
ORDER BY table_name, index_name, seq_in_index;

-- 3. 检查字符集
SELECT 
    TABLE_NAME,
    TABLE_COLLATION,
    COLUMN_NAME,
    CHARACTER_SET_NAME,
    COLLATION_NAME
FROM information_schema.COLUMNS
WHERE TABLE_SCHEMA = 'xzs'
AND CHARACTER_SET_NAME IS NOT NULL
LIMIT 100;
```
